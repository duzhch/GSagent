# P0-02 QC Risk Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the first practical deep-QC slice: genotype missingness stats, high-risk pre-run blocking, and auditable manual QC override.

**Architecture:** Extend `DatasetProfile` with a typed QC summary that can parse PLINK2 `.smiss/.vmiss` outputs when provided. Enforce a QC risk gate in `run_job` before workflow execution. Add a dedicated override API to clear the QC block with approver/reason audit trail and decision trace node.

**Tech Stack:** Python 3.10+, FastAPI, Pydantic, pytest
